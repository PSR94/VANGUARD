# Project Maintenance

This repo is maintained as a focused, production-minded solo project. The goal is not heavy process. The goal is to keep the work reviewable, the backlog believable, and the history understandable six months later.

Use this folder as the operating manual for the public GitHub surface:

- [Commit guidance](./commit-guidelines.md)
- [Label taxonomy](./labels.md)
- [Triage guide](./triage.md)
- [Review guidance](./review-guidelines.md)
- [Release checklist](./release-checklist.md)
- [Initial backlog](./initial-backlog.md)

The intent is simple:

- issues should describe real work
- PRs should explain behavior and risk
- commits should be readable without digging through the diff
- release prep should be boring and repeatable
